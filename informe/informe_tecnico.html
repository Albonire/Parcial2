1. Resumen ejecutivo

Este informe presenta el análisis interno del modelo ESM-2 (Evolutionary Scale Modeling), un transformer científico de tipo encoder-only diseñado para procesar secuencias de proteínas. El objetivo no fue entrenar ni ajustar el modelo, sino comprender y demostrar, mediante evidencia computacional, cómo una secuencia de aminoácidos se transforma internamente en tokens, embeddings iniciales, representaciones contextualizadas (hidden states), matrices de atención y salidas útiles para análisis de datos científicos.

Se utilizó el modelo facebook/esm2_t6_8M_UR50D, de aproximadamente 8 millones de parámetros, ejecutado sobre tres secuencias: una original, una mutada con un cambio puntual y una alterada por inversión. Se extrajeron tokens, identificadores, formas de last_hidden_state, hidden states y matrices de atención. Adicionalmente, se inspeccionó el código fuente de la implementación de ESM en la librería Hugging Face Transformers para identificar componentes arquitectónicos como el tokenizer, los embeddings, la self-attention, las capas feed-forward y la cabeza de masked language modeling.

Los resultados confirman que la secuencia mutada conserva una alta similitud coseno con la original, mientras que la secuencia alterada se aleja en el espacio de representación. El experimento de masked language modeling evidenció el carácter bidireccional del modelo. Finalmente, se discuten usos reales de ESM-2 en ciencia de datos aplicada a proteínas y sus limitaciones, destacando que la atención no constituye prueba de causalidad biológica.

2. Introducción

Los modelos de tipo transformer han transformado el procesamiento de secuencias en múltiples dominios, desde el lenguaje natural hasta la biología computacional. ESM-2 representa una aplicación científica de esta arquitectura, especializada en secuencias de aminoácidos. A diferencia de los modelos conversacionales, ESM-2 no genera texto ni responde preguntas: produce representaciones matemáticas de proteínas que pueden emplearse en tareas posteriores.

Este trabajo se enmarca en la comprensión profunda de un transformer preentrenado. Se busca abrir la llamada "caja negra" y explicar, con evidencia ejecutable, qué ocurre dentro del modelo cuando recibe una secuencia proteica. El informe documenta el proceso completo: desde la carga del modelo hasta la interpretación de sus salidas internas.

3. Marco teórico

3.1 El transformer original

El transformer fue propuesto por Vaswani et al. (2017) en el artículo Attention Is All You Need. Su innovación principal consistió en reemplazar la recurrencia y la convolución por mecanismos de atención, permitiendo procesar secuencias de forma paralela y capturar relaciones de largo alcance. La idea central es convertir cada unidad de información en un vector y calcular relaciones entre esas unidades mediante atención.

3.2 Familias de transformers

TipoCómo atiendeUso típicoEjemplos
Encoder-onlyAtención bidireccional: cada token mira a todosComprensión, clasificación, embeddings, MLMBERT, ESM-2
Decoder-onlyAtención causal: cada token mira solo hacia atrásGeneración autoregresivaGPT
Encoder-decoderEncoder comprende, decoder generaTraducción, resumenT5, BART

ESM-2 pertenece a la familia encoder-only, ya que procesa la secuencia completa de aminoácidos de forma bidireccional y produce representaciones contextualizadas, en lugar de generar una secuencia paso a paso como GPT (Devlin et al., 2018; Radford et al., 2018).

3.3 Tokens, embeddings y hidden states

Un transformer no procesa directamente aminoácidos en forma cruda. Primero convierte la entrada en tokens; en ESM-2, cada token suele corresponder a un aminoácido, junto con tokens especiales. Cada token se asocia a un identificador numérico (input_ids) y luego a un embedding inicial: un vector que ubica el token en un espacio matemático.

Diferencia crítica Embedding inicial: vector del token antes de pasar por las capas transformer.
Hidden state: representación contextualizada después de la atención y las capas; incorpora información de los demás tokens de la secuencia.

3.4 Self-attention y multi-head attention

El mecanismo de self-attention determina cuánto debe atender cada token a los demás. Para ello se calculan tres proyecciones: Query (Q), Key (K) y Value (V).

Attention(Q, K, V) = softmax( Q·KT / √dk ) · V
  • Q (Query): qué busca cada token.
  • K (Key): cómo se presenta cada token ante los demás.
  • V (Value): qué información aporta cada token.

En la atención multi-cabeza, el modelo ejecuta varias atenciones en paralelo. Cada cabeza puede capturar patrones distintos dentro de la secuencia.

3.5 Masked Language Modeling

El Masked Language Modeling (MLM) es un objetivo de entrenamiento donde se ocultan algunos tokens y el modelo debe predecirlos usando el contexto. BERT introdujo este enfoque para lenguaje natural (Devlin et al., 2018). ESM-2 aplica la misma idea sobre proteínas: oculta aminoácidos y predice los más probables, aprovechando contexto bidireccional.

4. ¿Qué es ESM-2?

ESM significa Evolutionary Scale Modeling. ESM-2 es un modelo de lenguaje de proteínas desarrollado por Meta AI/FAIR, que procesa secuencias de aminoácidos mediante transformers. No es un chatbot: su entrada son secuencias proteicas y sus salidas son representaciones matemáticas, hidden states, logits para MLM y, si la implementación lo permite, matrices de atención.

Definición operativa ESM-2 es un transformer encoder-only de proteínas que puede usarse como modelo de embeddings científicos. Arquitectónicamente es un transformer; funcionalmente se usa para obtener vectores por aminoácido o por proteína completa.

ESM-2 se relaciona con ESMFold, que aprovecha sus representaciones para predecir estructura tridimensional a partir de secuencia (Lin et al., 2023).

6. Metodología computacional

El procedimiento se implementó en Python usando PyTorch y la librería Hugging Face Transformers. El entorno se ejecutó tanto localmente (Fedora 44) como en Google Colab. No se requirió GPU debido al tamaño reducido del modelo.

  1. Carga del tokenizer y del modelo preentrenado.
  2. Tokenización de cada secuencia y extracción de input_ids.
  3. Inferencia con output_hidden_states=True y output_attentions=True.
  4. Extracción de last_hidden_state, hidden states y atenciones.
  5. Cálculo de embeddings globales y similitud coseno.
  6. Reducción dimensional mediante PCA.
  7. Visualización de matrices de atención.
  8. Experimento de masked language modeling con top-k.
Nota sobre atenciones Algunas implementaciones optimizadas no retornan mapas de atención. Por ello, el modelo se cargó con attn_implementation="eager" cuando estuvo disponible, para favorecer el retorno explícito de atenciones.

10. Limitaciones

Lo que no debe afirmarse No se debe afirmar que ESM-2 descubre medicamentos automáticamente, reemplaza experimentos de laboratorio o demuestra causalidad biológica mediante mapas de atención.
  • La atención muestra patrones estadísticos, no causalidad biológica.
  • El modelo refleja sesgos de los datos de entrenamiento.
  • Las predicciones requieren validación científica y, en muchos casos, experimentación.
  • Modelos más grandes requieren recursos de cómputo significativos.

11. Conclusiones

  1. ESM-2 es un transformer encoder-only aplicado a proteínas, no un modelo generativo.
  2. El last_hidden_state es una representación contextualizada, distinta del embedding inicial.
  3. La self-attention se basa en las proyecciones Q, K y V, identificables en el código fuente.
  4. La secuencia mutada conserva alta similitud con la original; la alterada se aleja.
  5. El masked language modeling evidencia el carácter bidireccional del modelo.
  6. ESM-2 tiene usos reales valiosos, pero con limitaciones claras de interpretación.

13. Anexos de código y salidas

Anexo A: Carga del modelo e inferencia

import torch
from transformers import AutoTokenizer, EsmModel

model_name = "facebook/esm2_t6_8M_UR50D"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = EsmModel.from_pretrained(model_name, attn_implementation="eager")
model.eval()

sequence = "MKTAYIAKQRQISFVKSHFSRQDILD"
inputs = tokenizer(sequence, return_tensors="pt")

with torch.no_grad():
    outputs = model(**inputs,
                    output_hidden_states=True,
                    output_attentions=True)

print("last_hidden_state:", outputs.last_hidden_state.shape)
print("hidden_states:", len(outputs.hidden_states))

Anexo C: Masked Language Modeling

from transformers import EsmForMaskedLM

mlm_model = EsmForMaskedLM.from_pretrained(model_name)
mlm_model.eval()

masked = "MKTAYIAKQRQISFVK".replace("Y", tokenizer.mask_token, 1)
inputs = tokenizer(masked, return_tensors="pt")
idx = (inputs["input_ids"] == tokenizer.mask_token_id).nonzero(as_tuple=True)[1]

with torch.no_grad():
    out = mlm_model(**inputs)

logits = out.logits[0, idx, :]
topk = torch.topk(logits, k=5, dim=-1)
for s, t in zip(topk.values[0], topk.indices[0]):
    print(tokenizer.convert_ids_to_tokens([t.item()])[0], float(s))
